1、 黑白名单测试（black-white-list_shpec.sh）

编号

测试命令

预期结果

说明

1

curl --interface 127.0.0.1 http://127.0.0.1/

curl返回52

源ip地址不在白名单，禁止访问

2

curl --interface 127.0.0.11 http://127.0.0.1/

curl返回0

源ip地址在白名单，允许访问

3

curl --interface 127.0.0.11 http://127.0.0.1:81/

curl返回52

源ip地址在黑名单，禁止访问

4

curl --interface 127.0.0.1 http://127.0.0.1:81/

curl返回0

源IP地址不在黑名单，允许访问

5、 HTTP路由测试（http-route_shpec.sh）

编号

测试命令

预期结果

说明

1

curl -s -i -H 'a:1' -H 'b:2' -H 'c:3' http://127.0.0.1/path/abc?def=2

返回页面

路由匹配 path、header、params 正常

2

curl -s -i -H 'a:1' -H 'b:2' -X DELETE http://127.0.0.1/path?def=2

返回失败页面

匹配method失败

3

curl -s -i -H 'a:1' -H 'b:2' http://127.0.0.1/path?abc=x1

返回失败页面

匹配params失败

4

curl -s -i http://127.0.0.1/path/abc/tail

返回页面

匹配多级path成功

 

6、 HTTPS 透传测试（https-passthrough_shpec.sh）

2

3

curl -s -i -k --http1.1 --resolve no.test.com:8443:127.0.0.1 https://no.test.com:8443/

curl 返回 35

没有匹配到域名，返回35错误码
