config.json配置字典

一、 全局配置（Configs）

编号

配置项名称

用途描述

可选值

1

EnableDebug

是否输出调试日志信息

true，false

2

StripAnyHostPort

是否忽略HTTP请求头中Host的端口号，比如：将 

Host: www.aaa.com:8080 

视为

Host: www.aaa.com 

true，false

3

DefaultPassthroughUpstreamPort

TLS Passthrough 场景，

如果没有配置上游端口，则使用此端口号

无默认值，

通常设置为 443

 

二、 监听端口配置（Listeners）

编号

配置项名称

用途描述

可选值

参考配置

1

Port

应用使用的端口号

1~65535

 

2

Listen

非必须配置，监听端口号，

如果未配置，使用 Port 为监听端口号

1~65535

 

3

Protocol

端口所使用的协议

HTTP、HTTPS、TLS、TCP

 

4

AccessControlLists

访问控制列表，设置访问者IP地址黑白名单。如果设置了白名单，就以白名单为准；如果未设置白名单，就以黑名单为准。

可以配置IP，也可以配置IP段

编号

子项

1

blacklist

2

whitelist

 

"AccessControlLists": {

   "blacklist": [

      "127.0.0.1",

      "192.168.122.1/24"

    ],

    "whitelist": [

      "192.168.122.18",

      "127.0.0.11/24"

    ]

 },

5

bpsLimit

网络限速（单位：字节/秒）

 

"bpsLimit": 10000000

6

TLS

配置 TLS相关的证书信息

编号

子项

可选值

说明

1

TLSModeType

Terminate

TLS

解码模式

Passthrough

TLS

通过模式

2

mTLS

true

启用客户端证书认证

false

不启用客户端证书认证

3

Certificates

编号

 

子项

说明

1

CertChain

证书

2

PrivateKey

私钥

3

IssuingCA

签名证书

 

 

"TLS": {

  "TLSModeType": "Terminate",

  "mTLS": false,

  "Certificates": [

    {

      "CertChain": "-----BEGIN CERTIFICATE-----......",

      "PrivateKey": "-----BEGIN RSA PRIVATE KEY-----......",

      "IssuingCA": "-----BEGIN CERTIFICATE-----......"

    }

  ]

}

编号

配置项名称

用途描述

编号

子项

说明

ConnectionSettings

熔断配置，主要用于 HTTP访问

编号

子项

说明

1

MinRequestAmount

触发熔断必须达到的访问次数 

2

StatTimeWindow

熔断判定统计窗口

3

SlowTimeThreshold

慢访问时间阈值，单位：秒

4

SlowAmountThreshold

触发熔断的慢访问次数

5

SlowRatioThreshold

触发熔断的慢访问占比

6

ErrorAmountThreshold

触发熔断的失败访问次数

7

ErrorRatioThreshold

触发熔断的失败访问占比

8

DegradedTimeWindow

熔断时间窗口

9

DegradedStatusCode

熔断时返回的HTTP状态码

10

DegradedResponseContent

熔断时返回的提示信息

 

 

 

 

 

 

 

"ConnectionSettings": {

        "http": {

          "CircuitBreaking": {

            "MinRequestAmount": 10,

            "StatTimeWindow": 60,

            "SlowTimeThreshold": 1.0,

            "SlowAmountThreshold": 10,

            "SlowRatioThreshold": 0.3,

            "ErrorAmountThreshold": 10,

            "ErrorRatioThreshold": 0.15,

            "DegradedTimeWindow": 60,

            "DegradedStatusCode": 409,

            "DegradedResponseContent": "Coming soon .... "

          }

      }

  }

RetryPolicy

 }

UpstreamCert

访问上游服务，

            "PrivateKey": "-----BEGIN RSA PRIVATE KEY-----......",

            "IssuingCA": "-----BEGIN CERTIFICATE-----......"

}

StickyCookieName

HTTP Cookie名称

用于负载均衡粘滞

7

StickyCookieExpires

HTTP Cookie有效期

单位：秒

 

"StickyCookieExpires": 3600,

8

HealthCheck

对服务上游进行健康检测

编号

子项

说明

1

Interval

主动检测的时间间隔，

如果不设置，不会进行主动健康检测

单位：秒

2

MaxFails

连续失败次数后，将上游设置为不可用

3

FailTimeout

被动检测时，服务不可用时间间隔。当连续失败次数达到 MaxFails 后，该上游将暂时禁用。

如果设置了主动检查，该参数被忽略。

单位：秒

4

Uri

使用HTTP方式进行健康检查，为 HTTP path

如果未设置 Uri，表示用 TCP 方式进行端口健康检查

5

Matches

HTTP 健康检查时的判定条件

编号

子项

说明

1

Type

status：http 状态码

body: http 内容

Headers：http 头

2

Value

对应的值

3

Name

http 头字段名

 

 

 

 

 

"HealthCheck": {

        "Interval": 10,

        "MaxFails": 3,

        "FailTimeout": 30,

        "Uri": "/",

        "Matches": [

          {

            "Type": "status",

            "Value": "200"

          },

          {

            "Type": "body",

            "Value": "apache2.conf"

          },

          {

            "Type": "headers",

            "Name": "Server",

            "Value": "Apache/2.4.41 (Ubuntu)"

          }

        ]

      }
